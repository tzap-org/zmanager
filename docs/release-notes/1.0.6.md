# ZManager CLI 1.0.6 Release Notes

ZManager CLI 1.0.6 refreshes the archive engine for the desktop release path,
including current TZAP crates, Apple Archive coverage, and backend portability
fixes.

## Highlights

- Adds Apple Archive read support through the `zmanager-apple-archive` backend.
- Uses TZAP index metadata for faster browser/listing paths, including modified
  time display when TZAP metadata is present.
- Updates TZAP RootAuth trust handling to the production root.
- Fixes TZAP X.509 trust option initialization through the C ABI.
- Skips byte estimation for excluded directories to avoid misleading progress
  work.
- Fixes libarchive entry mode portability across supported targets.
- Pins `zmanager-core` to crates.io `tzap` dependencies for downstream desktop
  release builds.

Verify downloads with `SHA256SUMS` before installing.

## Known Backend Limits

- Creation is intentionally limited to ZIP, TAR.ZST, TZAP, and 7z.
- Editing an existing archive in place is not supported. Extract, modify, and
  create a new archive instead.
- RAR creation is not supported.
- Raw single-file streams extract to one output file; they are not archive
  containers.
- `--to-stdout` is for selected regular file data, not directories, links, or
  special entries.
- macOS release archives may depend on platform runtime libraries listed in the
  corresponding `zm-<target>.deps.txt` report. Linux release archives are
  statically linked and their reports must not contain ELF `NEEDED` entries.
