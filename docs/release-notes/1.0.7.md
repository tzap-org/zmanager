# ZManager CLI 1.0.7 Release Notes

ZManager CLI 1.0.7 provides stabilization fixes and enhancements for the desktop release path, focusing on Windows compatibility, CLI documentation alignment, progress tracking, and metadata preservation.

## Highlights

- **Windows Portability & FFI**: Added build stubs and resolved linking issues for Windows, improving compilation and FFI reliability.
- **Command Line Consistency**: Standardized doctor command help text and autocompletion wording across all supported shells.
- **Progress Reporting**: Complete implementation of phase-aware job progress tracking, batching progress events, and skipping byte estimation for excluded directories.
- **Metadata Restoration**: Corrected metadata flags and restoration logic for Apple Archive, 7z, ZIP, and TZAP backends.
- **Dependency Pinning**: Updated and pinned `tzap` crates to upstream revisions for reliable downstream desktop builds.

## Known Backend Limits

- Creation is intentionally limited to ZIP, TAR.ZST, TZAP, 7z, and TGZ.
- Editing an existing archive in place is not supported. Extract, modify, and
  create a new archive instead.
- RAR creation is not supported.
- Raw single-file streams extract to one output file; they are not archive
  containers.
- `--to-stdout` is for selected regular file data, not directories, links, or
  special entries.
- macOS release archives may depend on platform runtime libraries listed in the
  corresponding `zm-<target>.deps.txt` report. Linux release archives are
  statically linked and their reports must not contain ELF `NEEDED` entries.
